# 养老金政策参数表

## 一、计发月数表（N_TABLE）

### 1.1 完整N表

计发月数（N值）是国家统一规定的，用于计算个人账户养老金，计算公式为：

```
个人账户养老金 = 退休时个人账户余额 ÷ N值（计发月数）
```

| **51岁** | **183** | 女工人延迟1年 | 女工人 |

### 1.2 N值规律分析

```
年龄区间      N值递减规律
50-54岁:     平均每年递减 11-12个月
55-59岁:     平均每年递减 9-10个月
60-63岁:     平均每年递减 6-7个月

趋势: 年龄越大，N值递减越慢
```

### 1.3 N值经济学意义

**N值代表什么？**
- N值是国家精算确定的"平均预期领取月数"
- 年龄越大，N值越小，因为预期领取时间越短
- N值越小，月养老金越高（同样的账户余额）

**示例对比：**
```
假设退休时账户余额: 100,000元

50岁退休: 100,000 ÷ 195 = 512.82元/月
55岁退休: 100,000 ÷ 139 = 719.42元/月 (↑40.3%)
60岁退休: 100,000 ÷ 93  = 1,075.27元/月 (↑109.7%)
63岁退休: 100,000 ÷ 73  = 1,369.86元/月 (↑167.2%)
```

### 4.2 关键计算公式

```
┌────────────────────────────────────────┐
│          月养老金计算公式               │
├────────────────────────────────────────┤
│                                        │
│  月养老金 = 基础养老金 + 个人账户养老金  │
│                                        │
│  其中:                                 │
│  • 基础养老金 = 社平工资 × 1% × 缴费年限│
│                = 12,049 × 1% × N年     │
│                                        │
│  • 个人账户养老金 = 账户余额 ÷ N值      │
│                                        │
│  • N值 = 根据退休年龄查N_TABLE表        │
│                                        │
└────────────────────────────────────────┘
```

## 六、重要提醒

### 6.1 政策变化风险

⚠️ **请注意：**
1. 本表格基于2025年现行政策
2. 国家可能调整退休年龄政策
3. N值可能随精算结果调整
4. 社保参数每年可能变化

### 6.2 使用建议

✅ **建议：**
1. 定期更新 `cityConfig.js` 中的参数
2. 关注当地社保局政策通知
3. 计算结果仅供参考，实际以社保部门核定为准
4. 延迟退休需结合个人健康状况和职业规划

### 6.3 数据来源

📌 **参数来源：**
- N值：国家统一规定（国发〔1978〕104号等）
- 退休年龄：《国务院关于工人退休、退职的暂行办法》
- 北京参数：北京市人力资源和社会保障局2025年公布数据
- 程序实现：`miniprogram/config/cityConfig.js`
